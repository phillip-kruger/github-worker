github-worker

An AI-powered GitHub automation tool for any dev team

The Problem

  • 📬 Too many issues assigned, not enough hours
  • 🔄 Review requests pile up across repos
  • 🤖 Repetitive work: clone, fix, test, PR, review, CI, repeat
  • ⏰ Context switching between issues kills productivity

The Solution

A House Elf that works while you sleep

  • 🧠 Uses Claude Code CLI with full MCP server access
  • 🤝 Two GitHub accounts — you + bot
  • 📋 Multi-step stateful workflow — one step per scheduled run
  • 👀 You stay in control — approve, review, redirect

Architecture

Worker

  • Java / JBang
  • Runs every 5 minutes (configurable)
  • Shells out to gh and claude
  • State persisted in JSON
  • Git worktrees for isolation

Dashboard

  • Quarkus web app
  • Live updates via WebSocket
  • Trigger runs, view logs
  • Config editor
  • Topic-based discovery

Issue Workflow

How an issue goes from assigned to merged

Step 1 — Signal Intent

SignalMeaning
Self-assign an issueWorker picks it up
👀 on issue assigned by othersConfirms you want help

Step 2 — Bot Analyzes

NEW → APPROVAL → CODING → ...
  • Security triage — flags malicious issues
  • Bot posts a comment summarizing its understanding
  • Proposes a solution approach
  • Mentions you: "@you React with 👍 to proceed"

Step 3 — You Approve

NEW → APPROVAL → CODING → ...
  • 👍 — proceed to coding
  • 👎 — bot re-analyzes with your feedback comment
  • Up to 3 attempts, then gives up gracefully

Step 4 — Bot Codes

NEW → APPROVAL → CODING → REVIEW → ...
  • Creates a git worktree (fast, isolated)
  • Claude Code with full MCP access:
    • Quarkus Agent — build, test, extensions
    • All configured skills
  • Pushes to bot's fork, creates draft PR

Step 5 — Bot Reviews Itself

CODING → SELF_REVIEW → FIX_REVIEW → READY
  • Correctness, tests, documentation
  • Security, maintainability
  • Quarkus patterns and conventions
  • If issues found → fixes them → squashes → marks ready

Step 6 — Your Review

FIX_REVIEW → READY → SQUASH → CI
  • Bot adds you as reviewer, marks PR ready
  • You comment → bot addresses feedback automatically
  • Request changes → bot fixes and re-requests review
  • You approve → bot squashes to one commit

Step 7 — CI Monitoring

SQUASH → CI_WATCH → DONE
  • Bot monitors CI checks every 5 minutes
  • If CI fails → bot fixes and re-pushes
  • Up to 3 fix attempts
  • CI green → DONE

Full Issue Flow

NEW→ APPROVAL→ CODING→ SELF_REVIEW→ FIX_REVIEW→ READY→ FEEDBACK→ SQUASH→ CI_WATCH→ CI_FIX→ DONE

Each arrow = one scheduled run (5 min). You only interact at APPROVAL and READY.

Review Workflow

When you're requested as a reviewer on a PR

NEW → REVIEW_POSTED → DONE
  • Signal: self-request as reviewer OR 👀 on the PR
  • Bot does a full review: [CRITICAL] [SUGGESTION] [NIT]
  • Posts the review as the bot account

Discovery

Find issues and PRs you should look at — on demand

  • Configure topics: GraphQL, Dev UI, OpenAPI, ...
  • Scoped to orgs or repos: quarkusio/quarkus, quarkiverse
  • Issues matching your topics (without linked PRs)
  • PRs from others matching your topics or where you're CC'd
  • Issues/PRs where you're mentioned
  • Loaded on demand — expand the panel in the dashboard
  • Quick actions: 👀 react or self-assign

Manual Tracking

Add any issue or PR directly — bypass all filters

  • Click + in the dashboard header
  • Paste a GitHub URL or owner/repo#123
  • Auto-detects issue vs PR, fetches the title
  • No lookback limit, no eligibility checks
  • Goes straight into the workflow as NEW

Safety

  • 🔒 Security triage — every issue scanned for malicious content before code runs
  • 🤖 Bot account — all actions clearly from the bot, never your account
  • 📝 Draft PRs — you always review before anything goes live
  • 🔁 Max 3 retries — bot gives up gracefully, doesn't spam
  • 🔐 Lock file — prevents overlapping runs
  • ⏰ Active hours — only runs during your work hours

Tech Stack

Worker

  • Java 17+ / JBang
  • gh CLI for GitHub API
  • claude -p for AI coding
  • Git worktrees + Maven isolation
  • Picocli, Jackson

Dashboard

  • Quarkus (REST + WebSocket + Scheduler)
  • Single-page HTML/CSS/JS
  • No frontend framework
  • Port 7478

Pluggable AI agent — Claude Code is default, but the CodingAgent interface supports others

Install


# Install the worker
jbang app install github-worker@House-elves/github-worker

# Interactive setup
github-worker --install

# Install the dashboard
jbang app install github-worker-ui@House-elves/github-worker
github-worker-ui --install

Works on Linux (systemd) and macOS (launchd). Survives restarts.

Configuration


# ~/.config/github-worker/config
GITHUB_USER=your-username
GITHUB_TOKEN=ghp_...
BOT_USER=your-username-bot
BOT_TOKEN=ghp_...
SCHEDULE=*/5 * * * *
TOPICS=GraphQL,Dev UI,OpenAPI
ORGS=quarkusio/quarkus,quarkiverse
ACTIVE_HOURS=07-22
EMAIL_NOTIFICATIONS=true
AGENT=claude

All editable from the dashboard UI

CLI Commands


# See what the worker would pick up
github-worker --preview

# Check tracked items
github-worker --status

# Process one item
github-worker --once

# Dry run (no writes)
github-worker --dry-run

# Run topic discovery (used by dashboard)
github-worker --discover

                          int timeoutSeconds) {
        // Analysis-only mode, no filesystem
    }
}

Set AGENT=my-agent in config
